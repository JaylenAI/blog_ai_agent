## 2-1. 경쟁사 / 유사 서비스 분석

블로그 자동화 영역의 주요 5개 도구와 본 프로젝트를 비교한다.

> ⚠️ **검수 요청**: 본 비교 대상이 마음에 들지 않거나 추가하고 싶은 도구(예: Hashnode AI, Frase, Surfer SEO)가 있으면 알려주세요.

### 핵심 관찰

1. **유료 SaaS** (Jasper, Copy.ai)는 영어 마케팅 콘텐츠 중심. 한국어 기술 블로그에는 부적합
2. **오픈소스 도구** (GPT-Researcher, CrewAI)는 강력하지만 **양식/배포는 직접 구현 필요**
3. **모든 도구가 다이어그램 자동 생성 미지원** → 이 프로젝트의 명확한 차별점
4. **모든 도구가 Tistory 자동 배포 미지원** → 한국 시장 진공 상태

## 2-2. SWOT 분석

```
   Strengths (강점)                                     Weaknesses (약점)
   ─────────────────                                     ─────────────────
   • Claude OAuth만 사용 → 편당 $0                       • Tistory 에디터 변경 시 자동화 깨질 위험
   • 한국어 기술 블로그 양식 100% 매칭                    • 양식이 흔들리면 모든 글이 어색해짐
   • Mermaid + HTML/CSS 자동화 (Tier 1)                  • 1인 운영 → 유지보수 책임 단독
   • Claude Code CLI Subagent 활용                        • Claude Code 의존 → 구독료 변경 위험
   • jaylenhan 실제 운영 데이터 보유                      • 자료조사 출처가 영어 의존 (한국어 자료 부족)
   • 6-8주 내 MVP 완성 가능                              • 양식 점검 자동화의 한계 (의미적 문제 못 잡음)

   Opportunities (기회)                                  Threats (위협)
   ─────────────────                                     ─────────────────
   • Anthropic Skills + MCP 생태계 급성장                • Tistory API 종료 후 Playwright 차단 가능성
   • Claude Code 구독자 한국 시장 확대 중                • 카카오 OAuth 보안 강화로 자동화 불가능
   • 한국 개발자 블로그 양식 정형화 → 확장 시장 존재      • Anthropic 모델 한국어 품질 변화
   • 부록 E 프로세스로 정식화된 산출물 → 포트폴리오 가치 • LLM "AI 슬롭" 평판 → 블로그 신뢰도 영향
   • Velog/Hashnode 어댑터로 확장 가능                   • Tistory 자동화 글 발견 시 운영자 조치
   • 오픈소스화 시 한국 개발자 커뮤니티 기여              • 글로벌 도구 (Notion AI 등) 한국 진출
```

### SWOT 매트릭스 → 전략 도출

| 전략 | 내용 |
|------|------|
| **SO (강점-기회)** | Claude Skills 생태계가 커지는 지금, 한국어 기술 블로그 자동화의 첫 production 사례 만들기 |
| **WO (약점-기회)** | Playwright 셀렉터 다중화 + Velog 어댑터를 동시 개발하여 Tistory 의존도 완화 |
| **ST (강점-위협)** | 검수 Gate 2개 + Critic 강제로 "AI 슬롭" 평판 회피. STYLE.md가 일관성 보장 |
| **WT (약점-위협)** | 자동화하지 않는 영역(검수, 주제 결정)을 명확히 선언 → 사람 중심 흐름 유지 |

## 2-5. 우리 프로젝트의 정체성 (한 문장)

> 🔥 **본 프로젝트는 "한국어 기술 블로그 양식 100% 자동 준수 + Tistory 자동 배포 + 편당 비용 $0"이라는 세 가지를 동시에 만족하는 유일한 도구다.**

3가지 차별점 중 하나라도 빠지면 시장에 이미 대안이 있다 (영문은 Jasper, 자료조사만 GPT-Researcher 등). 세 가지를 동시에 만족하는 도구는 없다 = 진공 시장.

---

## 2-6. 시장 진공의 이유 (왜 아직 없는가?)

이런 시장이 존재하는데 왜 아직 도구가 없는가? — 답을 의식적으로 기록한다.

| 이유 | 설명 |
|------|------|
| **타겟 시장이 작음** | 한국 기술 블로거 ~수만 명. 글로벌 SaaS의 ROI 낮음 |
| **Tistory API 부재** | 자동화 진입 장벽. 글로벌 회사는 한국 시장에 투자 안 함 |
| **양식이 사람마다 다름** | 일반화된 SaaS가 만들기 어려움 → 1인 운영 / OSS가 유리 |
| **Claude Skills + MCP 인프라가 2025년 후반에야 성숙** | 2024년에는 만들고 싶어도 도구가 없었음 |
| **한국어 LLM 품질이 2024년 후반에야 격식체 안정** | 그 전에는 어색한 결과물 |

→ 본 프로젝트가 **2026년 5월에 만들기 시작하는 것이 정확한 타이밍**.
