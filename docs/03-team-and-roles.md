# Phase 3 · 팀 빌딩 및 역할 분담

> 💡 부록 E Phase 3 "누가 뭘 하는지"가 불명확한 팀은 반드시 갈등이 생긴다. 1인 프로젝트라도 역할을 분리해서 사고한다.

---

## 3-1. 프로젝트 유형

**1인 프로젝트** — Jaylen H 단독 개발·운영.

단, 이 프로젝트는 Claude Code가 **하네스 런타임**이자 **AI 협업 파트너**로 기능하므로, 전통적인 1인 프로젝트와는 다르다. 역할을 "사람이 하는 것"과 "Claude가 하는 것"으로 명확히 분리한다.

---

## 3-2. 역할 분리 — 사람 vs Claude

### 🧑 사람 (Jaylen H) 담당 — "결정"과 "판단"

| 역할 | 담당 범위 | 산출물 |
|------|----------|--------|
| **PM** | 주제 선정, 마일스톤 관리, 우선순위 결정 | 일정표, 주간 회고 |
| **콘텐츠 디렉터** | Gate 1 아웃라인 승인, Gate 2 최종 승인 | 승인/수정 피드백 |
| **QA** | 생성된 글 품질 확인, 양식 최종 검수 | 검수 체크리스트 |
| **배포 관리자** | Tistory 로그인(수동), 게시 최종 버튼 | 발행된 글 URL |
| **양식 설계자** | STYLE.md 규칙 정의·갱신 | STYLE.md 버전 관리 |

### 🤖 Claude (하네스) 담당 — "실행"과 "자동화"

| 역할 | 담당 범위 | 산출물 |
|------|----------|--------|
| **Researcher** | 자료수집 (WebSearch/WebFetch/Context7) | `references/{slug}/*.md` |
| **Outliner** | 아웃라인 생성 (양식 9개 대섹션) | `02_outline.json` |
| **Writer** | 본문 작성 (STYLE.md 100% 준수) | `06_final.md` |
| **Imager** | Mermaid/SVG/차트 생성 | `04_images/*.png, *.svg` |
| **Critic** | 양식 검증 + AI 슬롭 제거 | `07_critique.json` |
| **Publisher** | 마크다운→HTML 변환, Tistory 입력 자동화 | `08_tistory.html` |

### 분업 원칙

```
사람이 결정한다 → Claude가 실행한다 → 사람이 검증한다
         ↑                                    ↓
         └────── 피드백 ──────────────────────┘
```

> ⚠️ **핵심 원칙**: Claude는 "무엇을 쓸지"를 결정하지 않는다. "어떻게 쓸지"를 실행한다. 이 경계가 흔들리면 AI 슬롭 양산기가 된다. ([[00-elevator-pitch#0-5|§0-5 작은 철학]])

## 3-3. 1인 프로젝트 주간 루틴 (부록 E §3-3)

```
월: PM 모드
    - 주간 주제 2~3개 선정
    - 마일스톤 진행률 체크
    - STYLE.md 규칙 위반 패턴 리뷰

화~목: 개발자 모드
    - 하네스 구현 (Skill/Subagent/Validator)
    - 파이프라인 테스트

금: QA + 운영 모드
    - 실제 블로그 글 1편 생성 (파이프라인 E2E 테스트)
    - 양식 검수 → 발행
    - 버그/개선점 기록

토: 문서화 모드
    - 이번 주 변경사항 docs/ 반영
    - STYLE.md 갱신 (필요 시)
    - 주간 회고 (4L)
```

## 3-4. 협업 규칙 (사람 ↔ Claude)

### 소통

- **주 소통 채널**: Claude Code CLI (이 프로젝트 디렉토리에서)
- **작업 기록**: Git 커밋 메시지 + `docs/meetings/` 회의록
- **긴급 결정**: CLAUDE.md §10 상황별 대응 따름

### 검수 게이트 (비협상)

- **Gate 1** (아웃라인): 사용자 승인 없이 본문 작성 진입 금지
- **Gate 2** (최종): 사용자 승인 없이 Tistory 게시 금지
- `--auto` 플래그: Gate 1만 건너뛸 수 있음. Gate 2는 절대 자동화 불가

### 갈등 해결 (양식 판단 불일치)

```
1. Claude가 양식 위반으로 판단 → 사용자에게 구체적 근거와 함께 보고
2. 사용자가 "이번엔 괜찮다"고 판단 → 그 글에 한정해 예외 허용
3. 사용자가 "앞으로도 이렇게"라고 판단 → STYLE.md 규칙 갱신 + ADR 기록
4. 모호한 경우 → oracle Subagent에 의견 요청 → 사용자 최종 결정
```
