# Phase 0 · 아이디어 발굴 및 주제 선정

> 💡 부록 E Phase 0 "어떤 주제를 할까?"에서 머무르는 시간이 가장 길다. 체계적으로 좁혀야 한다.

이 문서는 **이 프로젝트(blog_ai_agent)가 어떻게 선정되었는지**의 과정과 근거를 정리한다.

---

## 0-1. 아이디어 소스 (어디서 나왔는가)

| 소스 | 이 프로젝트에서의 적용 |
|------|------------------|
| **불편함에서 출발** ✅ | 블로그 글 1편 작성에 4-8시간 소요. 자료조사가 비효율적이고 양식 일관성이 흔들림 |
| **트렌드 키워드** ✅ | AI Agent + Skills + MCP가 2026년 핵심. 사용 안 하면 손해 |
| **기존 서비스 개선** ✅ | Jasper / Copy.ai 등은 영어 중심, 한국어 기술 블로그 자동화는 빈 시장 |
| **개인 운영 자산 활용** ✅ | `jaylenhan.tistory.com` 운영 데이터 + 자체 양식 표준 보유 |

→ 4개 소스가 동시에 가리키는 주제 = **한국어 기술 블로그 자동화 AI Agent**

## 0-2. 후보 아이디어 5기준 평가 (부록 E 형식)

부록 E의 5가지 기준에 따라 후보 아이디어 4개를 점수화했다.

| 아이디어 | 실현 | 차별 | 기술성장 | 포폴 | 사용자 | **합계** |
|---------|-----|-----|--------|-----|------|--------|
| **(A) 블로그 자동화 AI Agent** | 5 | 4 | 5 | 5 | 5 | **24 / 25** ⭐ |
| (B) 자체 LLM Fine-tuning 플레이그라운드 | 2 | 3 | 5 | 4 | 1 | 15 / 25 |
| (C) 모니터링 대시보드 (DataDog Lite) | 3 | 2 | 3 | 3 | 2 | 13 / 25 |
| (D) AI 코드 리뷰 봇 | 4 | 2 | 4 | 4 | 3 | 17 / 25 |

### 평가 근거 (왜 24점인가)

**🟢 실현 가능성 (5/5)**
- 핵심 도구 모두 보유: Claude Max 구독, Tistory 블로그, Claude Code CLI
- 외부 의존성 최소: WebSearch 내장 + Playwright + Mermaid (모두 무료)
- 1인 개발자 6-8주 분량으로 측정됨

**🟢 차별성 (4/5)**
- 한국어 기술 블로그 + Tistory 자동화는 시장에 거의 없음
- jaylenhan.tistory.com 양식 100% 매칭은 본인밖에 못 함
- (-1: 영어권에는 유사 도구 다수 — Jasper, Copy.ai)

**🟢 기술 성장 (5/5)**
- Claude Agent SDK + Skills 실전 운용 경험
- Multi-stage 파이프라인 설계 (Orchestrator + Subagent)
- Mermaid 자동 생성, Playwright 자동화, HTML/CSS 썸네일 등 광범위
- AI 슬롭 검증 (Critic + Reflection 패턴) 구현

**🟢 포트폴리오 가치 (5/5)**
- 실제 운영 중인 블로그(jaylenhan)에 적용 → 결과물이 살아있음
- AI Agent 시스템 설계 풀스택 경험 → 면접/이력서 STAR 사례
- 부록 E 16 Phase 모두 거치는 정식 프로세스 따른 프로젝트

**🟢 사용자 존재 (5/5)**
- 1차 사용자 = 본인 (즉시 dogfooding)
- 2차 = jaylenhan.tistory.com 구독자 / 같은 패턴으로 글 쓰는 한국 개발자
- 검증 사이클이 빠름 — 글 1편 만들고 발행하면 즉시 피드백

## 0-4. 엘리베이터 피치

### 📌 30초 버전 (대외용)

> **한국어 기술 블로그를 운영하는 개발자**를 위한 **자료조사부터 작성·이미지·배포까지 자동화하는 AI Agent**입니다.
> 기존 Jasper / Copy.ai 같은 영문 콘텐츠 도구와 달리 **한국어 기술 블로그 양식**과 **Tistory 자동 발행**에 특화되어 있고, **Claude Code 구독만으로 추가 비용 없이** 운영됩니다.

### 📌 15초 버전 (압축)

> Claude Code 안에서 "X 블로그 만들어줘" 한 줄로 1,500줄짜리 한국어 기술 블로그 글을 다이어그램 5장과 함께 자동 생성, Tistory에 자동 발행하는 1인 운영용 AI Agent.

### 📌 5초 버전 (해커톤급)

> "주제 → 블로그 자동 게시"를 Claude OAuth만으로, 편당 $0.

## 0-5. 이 프로젝트의 작은 철학

이 프로젝트가 다른 블로그 자동화 도구와 결정적으로 다른 지점:

> 🔥 **AI는 작가를 대체하지 않는다. 보일러플레이트를 제거한다.**

- 자동화의 대상 = 자료조사 수집, 양식 적용, 다이어그램 그리기, HTML 변환, Tistory 입력 같은 **반복적/기계적 작업**
- 자동화하지 않는 것 = **무엇을 쓸지 결정(주제)**, **아웃라인이 옳은지 판단(Gate 1)**, **최종 글이 충분한지 판단(Gate 2)**
- 검수 게이트 2개는 자동화의 약점이 아니라 **설계 의도**다. ([[CLAUDE|CLAUDE.md §8 검수 게이트 정책]])

이 철학이 흔들리면 "AI 슬롭 양산기"가 된다. 흔들지 않는다.
